2019-06-11
字数：153字 | 阅读本文大概需要 ：1分钟

生成签名证书文件

  1. 打开 DOS 窗口

  2. 运行如下命令：

    keytool -genkey -alias android.keystore -keyalg RSA -validity 36500 -keystore android.keystore

确认[否] y

  • -alias android.keystore 签名证书别名
  • -keyalg RSA 签名算法
  • -validity 36500 代表有效期天数
  • -keystore android.keystore 证书文件名
  1. 执行完成后，当前目录中会生成 android.keystore 文件

获取 SHA1 或 MD5

命令获取

  1. 进入证书文件所在目录，打开 DOS 窗口

  2. 运行命令

    keytool -list -v -keystore 证书文件名

  3. 输入密码 「123456」 即可

Android Studio「获取 debug 版签名」


android studio debug 签名文件路径示例

D:.AndroidStudio\avd.android\debug.keystore

  1. 安装 Oracle 时字符集一定要使用 AL32UTF8「一个汉字占 3 个字符长度」，避免导入到 zhs16gbk「一个汉字占 2 个字符长度」 字符集数据库时长度不够。

  2. zhs16gbk是中文字符集，也就是适合在中国用，只能存储中文和英文字符，如果你存储韩文则显示为乱码（没有编码），而al32utf8是utf8字符集，u是unicode的意思，适合中文、韩语、日语等等不同的语言使用。那么为什么我们要在中国使用zhs16gbk存储中文呢？这是因为utf8存储中文的效率不如zhs16gbk，比如一个字“懂”，zhs16gbk采用2个字符存储，而al32utf8采用3-4个字符存储，这样效率就有了高低之分。
